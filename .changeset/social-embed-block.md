Add support for the `social-embed` block (Twitter/X, Instagram, Facebook, TikTok, LinkedIn, Pinterest). The renderer follows the plugin's source priority — author-pasted `embedCode` → provider `oembed.html` → a graceful `<a>` link card (with author/thumbnail when available) — and emits the trusted embed markup verbatim inside an alignment-aware, accessible `<figure aria-label="{provider} post by {author}">` with an optional `<figcaption>`. Platform widget scripts (twitter/instagram/tiktok/pinterest/facebook) are loaded **lazily and deduped** via an IntersectionObserver: no third-party JavaScript loads until an embed nears the viewport, each script is injected at most once per page, and embeds are re-processed on `astro:page-load` for view-transition navigations. LinkedIn needs no script (self-contained `<iframe>`), and `loading="lazy"` is added to provider iframes that don't already declare it. The block is overridable through the `blocks['social-embed']` prop.
